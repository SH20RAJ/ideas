# 1. Overview

**Unstory.app** is a campus-first AI-powered social infrastructure that connects students through communities, projects, events, and opportunities within a verified university ecosystem. It evolves into a campus super-app integrating social, collaboration, and essential student services.

---

# 2. Vision

To become the digital operating system for student life — enabling meaningful connections, collaboration, and growth across campuses globally.

---

# 3. Problem Statement

Students rely on fragmented platforms (WhatsApp, Instagram, LinkedIn, email) for:
* Social connections
* Club activities
* Events
* Collaboration
* Opportunities

This leads to:
* Missed opportunities
* Fake or low-trust interactions
* Lack of structured campus ecosystem
* Poor discovery of people and events

---

# 4. Solution

A unified, verified campus platform that enables:
* Social discovery (people + communities)
* Collaboration (projects, teams)
* Campus engagement (events, clubs)
* Utility services (later stage super-app)

---

# 5. Target Users

### Primary
* College students (18–26)
* Freshers and newcomers
* Club members and organizers
* Student creators and builders

### Secondary
* College administration
* Campus brands
* Recruiters (future)

---

# 6. Core Features (MVP)

### 6.1 Verified Onboarding
* College email verification (.edu/.ac.in)
* Student ID verification (optional)
* Anti-fake profile system

### 6.2 Smart Profiles
* Interests, skills, goals
* Academic + social identity
* Project portfolio

### 6.3 Social Discovery
* Interest-based matching
* Campus feed (not algorithm-heavy dopamine feed)
* Suggested people & communities

### 6.4 Communities & Clubs
* Create/join communities
* Club dashboards
* Role-based management

### 6.5 Events System
* Create campus events
* RSVP system
* Smart recommendations

### 6.6 Collaboration Layer
* Find teammates for projects
* Hackathon matching
* Skill-based discovery

---

# 7. Advanced Features (Phase 2)

# 10. Go-To-Market Strategy

### Phase 1: Pilot Campus
* Launch in 1 college
* Student ambassadors
* Integrate with clubs
* Focus on events + communities

### Phase 2: Multi-Campus Expansion
* Expand to 5–10 colleges
* Network effects
* Referral loops

### Phase 3: Scale
* Pan-India rollout
* Partnerships with institutions

# 11. Business Model
* Freemium for students

* Campus brand promotions
* Event promotions

# 12. Competitive Advantage
* Verified campus-only network
* Real-world + digital integration
* Not dopamine-driven social media
* Built for collaboration, not just content

---

# 13. Key Metrics
* Daily active users (DAU)
* Community engagement rate
* Event participation
* Matches/collaborations created
* Retention (Day 7 / Day 30)

---

# 14. Risks & Mitigation

### Risk: Low adoption
**Mitigation:** Start with high-value use case (events + clubs)

### Risk: Network effect problem
**Mitigation:** Campus-wise rollout

### Risk: Fake profiles
**Mitigation:** Strict verification system
